# GeoJSON Overlay for USD Composer ― 仕様書

- 対象拡張: `spacedata.geojson_overlay`
- バージョン: 0.4.0
- 対象環境: NVIDIA Omniverse Kit 106 / 107 系（USD Composer）, Python 3.10

---

## 1. 概要と目的

本拡張は、GeoJSON で表現された地理空間データ（点・線・面）を USD Composer の 3D シーンへ
重畳表示することを目的とする。設計上の最重要要件は **既存 USD シーンの非破壊** であり、
地理データはすべて専用のサブレイヤー `GeoJSONOverlay.usda` に author され、
ベースとなるシーン（root レイヤー）には一切変更を加えない。

---

## 2. アーキテクチャ

### 2.1 モジュール構成

| モジュール | 役割 | USD 依存 |
|---|---|:--:|
| `extension.py` | `omni.ext.IExt` 実装。メニュー登録、ウィンドウ生成・破棄。 | あり |
| `window.py` | `omni.ui` の操作ウィンドウ。入力値を収集し `UsdOverlayBuilder` を駆動。 | あり |
| `usd_builder.py` | GeoFeature → USD prim 変換、レイヤー管理、マテリアル色分け、PointInstancer。 | あり |
| `coordinate.py` | 座標変換（simple / pyproj）。 | なし |
| `geojson_loader.py` | GeoJSON 解析・正規化。 | なし |
| `triangulate.py` | 穴/凹対応の ear-clipping 三角形分割。 | なし |
| `terrain.py` | 地形ドレープ用の高さサンプラ。`from_prim` のみ USD 依存。 | 一部 |

USD 非依存モジュール（`coordinate` / `geojson_loader` / `triangulate` / `terrain` のコア）は、
Kit を起動せずに単体テスト・CI 検証が可能である。

### 2.2 データフロー

```
GeoJSON (file / str / dict)
        │  geojson_loader.load_geojson
        ▼
GeoFeature[]  (Point / LineString / Polygon に正規化)
        │  UsdOverlayBuilder.build
        ├─ 原点決定 → coordinate.set_origin
        ├─ ジオメトリ変換 (座標変換 + 地形ドレープ + 三角形分割)
        ├─ マテリアル生成・バインド (ColorManager)
        └─ EditTarget = GeoJSONOverlay.usda へ author
        ▼
USD Stage  /World/GeoJSONOverlay/...
```

### 2.3 シーングラフ

```
/World
└─ /World/GeoJSONOverlay            (Scope, kind=group)
   ├─ /World/GeoJSONOverlay/Looks   (Scope)
   │  └─ mat_<category>             (Material + Shader=UsdPreviewSurface)
   ├─ <featureName>                 (Xform+Sphere / BasisCurves / Mesh)
   └─ /World/GeoJSONOverlay/Points  (PointInstancer, 大量点モード時)
      └─ Prototypes/proto_<cat>_<i> (Sphere)
```

## 3. GeoJSON 解析仕様（`geojson_loader.py`）

### 3.1 入力

- `FeatureCollection` / `Feature` / 生 geometry のいずれも受理。
- 入力ソースはファイルパス、JSON 文字列、Python dict のいずれも可（`load_geojson`）。

### 3.2 正規化

すべてのジオメトリを単一型 **Point / LineString / Polygon** に平坦化する。

| GeoJSON 型 | 正規化結果 |
|---|---|
| `Point` | Point ×1 |
| `MultiPoint` | Point ×N |
| `LineString` | LineString ×1 |
| `MultiLineString` | LineString ×N |
| `Polygon` | Polygon ×1（`coordinates` = [外周, 穴1, …]） |
| `MultiPolygon` | Polygon ×N |
| `GeometryCollection` | 内包ジオメトリを再帰的に平坦化 |

- 座標は `[lon, lat]` または `[lon, lat, alt]`。`alt` 省略時は 0。
- `properties` と `id` は各 `GeoFeature` に保持される。
- 未対応・未知の型は無視（MVP 方針）。

### 3.3 `GeoFeature`

```python
@dataclass
class GeoFeature:
    geom_type: str          # "Point" | "LineString" | "Polygon"
    coordinates: list       # 型に応じた座標構造
    properties: dict
    source_id: str
```

## 4. 座標変換仕様（`coordinate.py`）

### 4.1 座標系の取り決め

USD は **Y-up** とし、水平面を X-Z に取る。

- X = 東方向（経度方向）
- Z = 北方向（緯度方向）
- Y = 高さ（`alt` をメートルとして利用、地形ドレープ時は地形高さ）

### 4.2 原点

変換は原点（経度・緯度）を基準とした相対座標を生成する。原点は USD 原点 (0,0,0) に対応する。
`UsdOverlayBuilder.build` は、明示指定がなければ全フィーチャ座標の重心（`centroid_lonlat`）を原点に採用する。

### 4.3 実装と切替

| 実装 | 方式 | 依存 |
|---|---|---|
| `SimpleEquirectangular` | 等距円筒近似。原点緯度で経度方向を `cos` 補正。 | なし |
| `PyprojTransformer` | pyproj による正式投影。既定は原点経度から UTM ゾーン自動選択、`epsg` 指定可。 | pyproj |

簡易方式の換算式（原点 (lon0, lat0)）:

```
x = (lon - lon0) * 111320 * cos(radians(lat0))
z = (lat - lat0) * 111320
y =  alt - alt0
```

`make_transformer(prefer, epsg, auto_install)`:

- `prefer="simple"` … 常に簡易方式。
- `prefer="pyproj"` … pyproj を使用。未導入時は `ensure_pyproj()` で pipapi 取得を試み、失敗時は簡易方式へフォールバック。
- `prefer="auto"`（既定）… pyproj が import 可能なら pyproj、無ければ簡易方式（自動インストールはしない）。

数百 m〜数十 km 程度の局所領域であれば、簡易方式でも実用上十分な精度を持つ。

## 5. ジオメトリ変換仕様（`usd_builder.py`）

### 5.1 種類別マッピング

| 地物 | USD prim | 補足 |
|---|---|---|
| Point | `Xform` + 子 `Sphere`(`marker`) | Xform に translate、Sphere に `radius` / `extent`。 |
| LineString | `BasisCurves`（`linear`） | `curveVertexCounts` / `points` / `widths`(vertex 補間)。 |
| Polygon（平面） | `Mesh` | ear-clipping による三角形群、`doubleSided=true`。 |
| Polygon（押し出し） | `Mesh` | 底面・上面キャップ ＋ 側壁（外周＋各穴の境界辺）。 |

### 5.2 押し出し仕様

- `extrude=True` で有効化。
- 高さの決定: `properties[height_field]`（既定キー `height`）に数値があれば優先、無ければ `extrude_height`。
- 底面 Y は座標変換（地形ドレープ含む）の結果、上面 Y は `底面 + 高さ`。
- 上下キャップは `triangulate` の結果を用い、側壁は外周および各穴リングの境界辺ごとに四角形を生成。
- `doubleSided=true`（裏面カリングを回避）。

### 5.3 三角形分割仕様（`triangulate.py`）

- 入力: `[外周, 穴1, …]`（各リングは (x,y,z)、閉じ重複は自動除去）。geometry 判定は XZ 平面で行い、Y は保持。
- 手順:
  1. 前処理（`_clean_ring`）で閉じ重複・連続重複点を除去。
  2. 外周を CCW、穴を CW に整える。退化リング（3 頂点未満・面積≒0）はスキップ。
  3. 穴を外周へブリッジ（O'Rourke の可視頂点法：右方向レイキャスト ＋ reflex 頂点判定）。
  4. ear-clipping で三角形分割。耳が見つからない退化時は「最大面積の凸頂点を強制クリップ」して前進。
- 計算量は概ね O(n²)。MVP 規模の地物には十分。
- 既知の制限: 自己交差ポリゴン、穴同士の接触などの退化形は厳密対応外。

### 5.4 属性保持仕様

各地物 prim に以下を付与する。

- typed custom 属性 `geojson:<key>`: 値の型を bool/int/double/string で自動判別。
- `customData["geojson:properties"]`: properties 全体（ラウンドトリップ用）。
- `customData["geojson:id"]`, `customData["geojson:geomType"]`。

### 5.5 マテリアル / 色分け仕様（`ColorManager`）

- `enable_materials=True` のとき、`properties[color_field]`（既定 `category`）ごとに
  `UsdPreviewSurface` マテリアルを `/World/GeoJSONOverlay/Looks` 配下に生成し、`MaterialBindingAPI` でバインド。
- 色の決定:
  - `color_map` に明示色があれば優先（`"#RRGGBB"` または `[r,g,b]`(0..1)）。
  - 未定義の category には 10 色パレットを巡回割り当て。
- `primvars:displayColor` も併せて設定（簡易表示・マテリアル未評価時の保険）。

### 5.6 PointInstancer 仕様

- 点数が `point_instancer_threshold`（既定 256）以上、または `force_point_instancer=True` で集約モード。
- `/World/GeoJSONOverlay/Points` に `PointInstancer` を 1 つ生成。
  - `positions`: 各点のローカル座標。
  - `protoIndices`: 各点の category に対応するプロトタイプ index。
  - `Prototypes/proto_<cat>_<i>`: category ごとの Sphere プロトタイプ（マテリアル/`displayColor` をバインド）。
- per-point `properties` を `primvars:geojson:<key>`（interpolation=`varying`）として公開（`instance_primvars=True`）。
  - 型は float / int / string を自動判別（`float[]` / `int[]` / `string[]`）。欠損は既定値で補完。
- properties 全体は `customData["geojson:propertiesList"]` にも一括保持。

### 5.7 地形ドレープ仕様（`terrain.py`）

- `height_sampler` を渡すと、各地物のローカル座標 (x,z) に対し地形高さ Y をサンプリングし、
  `Y = sample(x,z) + drape_offset` に置換する（サンプル不能な範囲外では元の Y を維持）。
- サンプラ:
  - `ConstantHeightSampler(y)`: 一定高さ。
  - `CallableHeightSampler(fn)`: 任意関数 `f(x,z)->y`。
  - `MeshHeightSampler(triangles)`: 三角形群に対し XZ 内包判定 ＋ バリセントリック補間で Y を返す。
    多数三角形向けに XZ 一様グリッドで空間インデックス化。複数該当時は最大 Y（上面）を採用。
- `MeshHeightSampler.from_prim(stage, path)`: USD メッシュ prim を読み、ワールド座標へ変換して構築。
  n>3 の面は扇状に三角形化。
- 押し出しと併用すると、底面が地形に沿い、上面は `底面 + 高さ`（建物が地形に乗る）。

## 6. レイヤー管理仕様

- 対象レイヤー名は設定 `exts."spacedata.geojson_overlay".overlay_layer_name`（既定 `GeoJSONOverlay.usda`）。
- root レイヤーの `subLayerPaths` を走査し、名前一致のサブレイヤーがあれば再利用。
- 無ければ生成し、root の `subLayerPaths` 先頭（= 最も強い合成順）に挿入。
  - root がファイル保存済みなら同ディレクトリにファイルレイヤーを作成、未保存なら匿名レイヤー。
- すべての author は `Usd.EditContext(stage, Usd.EditTarget(overlay_layer))` 内でのみ実行。
- `build(clear=True)` 時、`/World/GeoJSONOverlay` をいったん除去して再生成。
- `clear_overlay()` で重畳分のみ削除（ベースシーンは無傷）。

## 7. API リファレンス（主要）

### 7.1 `geojson_loader`

- `load_geojson(source) -> list[GeoFeature]`
- `normalize(geojson_dict) -> list[GeoFeature]`
- `centroid_lonlat(features) -> (lon, lat)`

### 7.2 `coordinate`

- `make_transformer(prefer="auto", epsg=None, auto_install=True) -> CoordinateTransformer`
- `pyproj_available() -> bool` / `ensure_pyproj(auto_install=True) -> bool`
- `SimpleEquirectangular` / `PyprojTransformer`（`set_origin`, `to_local`）

### 7.3 `triangulate`

- `triangulate(rings) -> (points, triangles)`
- `signed_area_xz(ring) -> float`

### 7.4 `terrain`

- `HeightSampler`（基底, `sample(x,z)->float|None`）
- `ConstantHeightSampler(y)` / `CallableHeightSampler(fn)`
- `MeshHeightSampler(triangles, grid=64)`、`from_points_indices(...)`、`from_prim(stage, path, grid=64)`

### 7.5 `usd_builder.UsdOverlayBuilder`

コンストラクタ引数（抜粋）:

| 引数 | 既定 | 説明 |
|---|---|---|
| `transformer` | auto | 座標変換器 |
| `point_radius` / `line_width` | 5.0 / 2.0 | Point 半径・線幅 |
| `extrude` / `extrude_height` / `height_field` | False / 20.0 / "height" | 押し出し設定 |
| `enable_materials` / `color_field` / `color_map` | True / "category" / None | 色分け設定 |
| `point_instancer_threshold` / `force_point_instancer` / `instance_primvars` | 256 / None / True | PointInstancer 設定 |
| `height_sampler` / `drape_offset` | None / 0.0 | 地形ドレープ設定 |

メソッド:

- `build(features, origin=None, clear=True) -> {"Point":n, "LineString":n, "Polygon":n}`
- `clear_overlay()`

補助:

- `ColorManager(stage, looks_path, palette, color_map)`、`parse_color(value)`

## 10. 制限事項・今後

- 三角形分割は自己交差・穴同士の接触などの退化形を厳密に扱わない。
- 押し出しは鉛直一様（傾斜壁・地形断面追従は未対応）。
- PointInstancer 化時は per-point の typed 属性を持たず、`primvars` と `customData` に集約する。
- 選択時のプロパティパネル連携、法線/UV 付与、マテリアルのテクスチャ対応は今後の拡張候補。
