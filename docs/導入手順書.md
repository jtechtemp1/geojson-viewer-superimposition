# GeoJSON Overlay for USD Composer ― 導入手順書

- 対象拡張: `spacedata.geojson_overlay`（バージョン 0.4.0）
- 対象環境: NVIDIA Omniverse Kit 106 / 107 系（USD Composer）, Python 3.10

本書は、拡張のインストールから有効化、`pyproj` の導入、基本的な使い方、
トラブルシュート、アンインストールまでを説明する。

---

## 1. 前提条件

- USD Composer（Kit 106 / 107 系）がインストール済みであること。
- kit-app-template を利用する場合は、クローン済みのプロジェクトがあること。
- 正式投影（UTM / Web Mercator）を使う場合のみ `pyproj>=3.4`。
  未導入でも簡易方式（等距円筒近似）で動作するため必須ではない。

---

## 2. 拡張の配置

配置方法は 2 通りある。いずれか一方でよい。

### 方法 A: kit-app-template に同梱する（推奨）

1. 本フォルダ `spacedata.geojson_overlay` を、テンプレートの拡張置き場へコピーする。

   ```
   <kit-app-template>/source/extensions/spacedata.geojson_overlay/
   ```

2. アプリの `.kit` ファイル、またはビルド設定で拡張が探索対象に含まれることを確認する
   （`source/extensions` は通常デフォルトで探索対象）。

### 方法 B: 拡張検索パスに追加する

1. 本フォルダを任意の場所に置く（例: `D:/exts/spacedata.geojson_overlay`）。
2. USD Composer の **Window → Extensions** を開き、歯車（⚙）→ **Extension Search Paths** に
   親フォルダ（例: `D:/exts`）を追加する。

---

## 3. 有効化

1. USD Composer を起動。
2. **Window → Extensions** を開く。
3. 検索欄に `spacedata.geojson_overlay` または「GeoJSON」と入力。
4. 一覧から **GeoJSON Overlay for USD Composer** を見つけ、トグルを **ON**。
5. 常に有効化したい場合は **AUTOLOAD**（自動起動）も ON にする。

有効化に成功すると、メニュー **Window** に **GeoJSON Overlay** が追加される。

---

## 4. `pyproj` の導入（任意）

正式投影を使う場合のみ。以下のいずれかの方法で導入する。失敗しても拡張は簡易方式で動作する。

### 方法 1: 拡張による自動取得（推奨）

`config/extension.toml` に次の宣言が入っており、オンライン接続があれば有効化時に Kit が解決を試みる。

```toml
[python.pipapi]
requirements = ["pyproj>=3.4"]
use_online_index = true
```

加えて、操作ウィンドウの「変換方式」で **pyproj** を明示選択すると、未導入時に
`omni.kit.pipapi` 経由でランタイム取得を試みる（内部関数 `coordinate.ensure_pyproj()`）。

### 方法 2: Kit の Python へ手動 pip

Kit 同梱の Python に対してインストールする。

```bash
# Linux / macOS
<kit>/python.sh -m pip install "pyproj>=3.4"

# Windows
<kit>\python.bat -m pip install "pyproj>=3.4"
```

### 方法 3: 社内ミラー / オフライン

`config/extension.toml` の `[python.pipapi]` に index を追加する。

```toml
[python.pipapi]
requirements = ["pyproj>=3.4"]
extra_args = ["--index-url", "https://your-mirror/simple"]
```

導入確認: 操作ウィンドウの「変換方式」で pyproj を選択し、重畳実行後のステータスに
`pyproj` と表示されれば有効。

## 5. 基本的な使い方

1. ベースとなる USD シーン（ステージ）を開く。
2. **Window → GeoJSON Overlay** でウィンドウを開く。
3. **GeoJSON** 欄でファイルを指定する（**参照…** ボタン、またはパス直接入力）。
   動作確認には同梱の `data/sample.geojson`（東京駅周辺）が使える。
4. 必要に応じて以下を設定する。

   | 項目 | 説明 |
   |---|---|
   | 変換方式 | auto / simple（依存なし）/ pyproj（正式投影） |
   | 原点自動 | ON で全フィーチャ重心を原点に。OFF で lon/lat を手入力 |
   | Point 半径 / 線幅 | マーカー球の半径、線の太さ |
   | 押し出し / 高さ / 高さ属性 | Polygon を立体化。高さは属性キー優先 |
   | 色分け / 属性キー | category 等で色分け |
   | 色マップ | `{"zone":"#E04C3D","route":[0.2,0.6,0.86]}` 形式の JSON |
   | 点の集約 | auto / 常に PointInstancer / 個別 prim |
   | 地形ドレープ / 地形prim / オフセット | 地形メッシュへ高さ吸着（第 6 章） |

5. **重畳表示** を押す。`/World/GeoJSONOverlay` 配下に prim が生成され、
   `GeoJSONOverlay.usda` レイヤーへ保存される。ステータスに生成数（Point / LineString / Polygon）が表示される。
6. やり直す場合は **Overlay をクリア** で重畳分のみ削除（元シーンは無傷）。

---

## 6. 地形ドレープの設定

地物を地形メッシュの高さに沿わせる手順。

1. ステージに地形メッシュ（`UsdGeom.Mesh`）を用意し、その prim パスを控える（例: `/World/Terrain`）。
2. 操作ウィンドウの **地形ドレープ** を ON。
3. **地形prim** にメッシュの prim パスを入力。
4. 必要なら **オフセット**（地形からの浮かせ量, m）を設定。
5. **重畳表示** を実行。各地物の Y が地形面に吸着する。
   押し出し ON の Polygon は、底面が地形に沿い上面が `底面 + 高さ` になる。

地形メッシュが見つからない場合は、ステータスに警告が表示され、ドレープは自動的に無効化される。

---

## 7. 生成物の確認

- **Stage** ウィンドウ: `/World/GeoJSONOverlay` 配下に地物 prim、`Looks` にマテリアルが見える。
- **Layer** ウィンドウ: `GeoJSONOverlay.usda` がサブレイヤーとして root の先頭に入っている。
- **Property** パネル: 地物 prim を選択すると `geojson:*` 属性が確認できる。
- 大量点モードでは `/World/GeoJSONOverlay/Points`（PointInstancer）が 1 つ生成される。

`GeoJSONOverlay.usda` は通常のサブレイヤーなので、ベースシーンと分けて保存・共有・破棄できる。

---

## 8. トラブルシュート

| 症状 | 原因 / 対処 |
|---|---|
| メニューに項目が出ない | 拡張が有効化されていない。Extensions で ON にする。検索パスを再確認。 |
| 「ステージが開かれていません」 | 先に USD シーン（ステージ）を開く。 |
| pyproj を選んでも simple になる | pyproj 未導入。第 4 章の方法で導入。オフライン環境では方法 2 / 3。 |
| 何も表示されない | 原点が大きくずれている可能性。原点自動を ON にするか、ビューを地物位置へフィット。 |
| ポリゴンが想定外の形 | 自己交差・退化形の可能性。三角形分割の対象外ケース（仕様書 5.3）を参照。 |
| 地形に吸着しない | 地形prim パスが誤り、または対象が Mesh でない。Stage で型・パスを確認。 |
| 取り込みをやり直したい | **Overlay をクリア** 後に再実行。ベースシーンは影響を受けない。 |

---

## 9. アンインストール

- **一時的に無効化**: Extensions ウィンドウでトグルを OFF。
- **重畳データの削除**: 操作ウィンドウの **Overlay をクリア**、または `GeoJSONOverlay.usda` を
  サブレイヤーから外す / 削除する。
- **拡張の削除**: 配置した `spacedata.geojson_overlay` フォルダを削除、または検索パスから外す。

いずれの操作もベースとなる USD シーンには影響しない。
